# 病例批量导入功能使用指南

## 功能概述

病例批量导入功能允许管理员通过上传 CSV 文件快速导入大量病例数据，适用于系统初始化或历史数据迁移场景。

## CSV 文件格式要求

### 1. 文件编码
- 必须使用 **UTF-8** 编码
- 文件扩展名必须为 `.csv`

### 2. 表头格式（必须严格匹配）
```
检查号,患者匿名ID,性别,年龄,检查时间,检查部位,科室
```

### 3. 字段说明

| 字段名 | 是否必填 | 数据类型 | 格式要求 | 示例 |
|--------|---------|---------|---------|------|
| 检查号 | 是 | 字符串 | 唯一标识，不可重复 | EX20240301001 |
| 患者匿名ID | 是 | 字符串 | 保护隐私的匿名标识 | P123456 |
| 性别 | 否 | 字符串 | M（男）或 F（女） | M |
| 年龄 | 否 | 整数 | 0-150 | 45 |
| 检查时间 | 是 | 日期时间 | yyyy-MM-dd HH:mm:ss | 2024-03-01 10:30:00 |
| 检查部位 | 是 | 字符串 | 检查的身体部位 | 胸部 |
| 科室 | 否 | 字符串 | 申请科室名称 | 放射科 |

### 4. 示例数据

```csv
检查号,患者匿名ID,性别,年龄,检查时间,检查部位,科室
EX20240301001,P123456,M,45,2024-03-01 10:30:00,胸部,放射科
EX20240301002,P123457,F,38,2024-03-01 11:00:00,胸部,放射科
EX20240301003,P123458,M,52,2024-03-01 14:20:00,胸部,急诊科
```

## API 接口使用

### 接口地址
```
POST /api/cases/import
```

### 请求参数
- **Content-Type**: `multipart/form-data`
- **file**: CSV 文件（必填）

### 请求示例（使用 curl）
```bash
curl -X POST http://localhost:8080/api/cases/import \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -F "file=@/path/to/your/cases.csv"
```

### 响应格式

#### 成功响应
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "totalRows": 100,
    "successCount": 95,
    "failedCount": 5,
    "errors": [
      {
        "row": 10,
        "reason": "检查号已存在"
      },
      {
        "row": 25,
        "reason": "必填字段缺失：检查时间"
      }
    ]
  }
}
```

#### 字段说明
- `totalRows`: CSV 文件中的总行数（不包括表头）
- `successCount`: 成功导入的病例数量
- `failedCount`: 导入失败的病例数量
- `errors`: 错误详情列表
  - `row`: 错误所在行号（从1开始，不包括表头）
  - `reason`: 错误原因描述

## 常见错误及解决方案

### 1. 文件格式错误
**错误信息**: "仅支持 CSV 格式文件"

**解决方案**: 
- 确保文件扩展名为 `.csv`
- 不要使用 Excel 的 `.xlsx` 格式

### 2. 表头格式错误
**错误信息**: "CSV 文件格式错误，期望表头：检查号,患者匿名ID,性别,年龄,检查时间,检查部位,科室"

**解决方案**:
- 检查表头是否完全匹配（包括逗号和顺序）
- 确保没有多余的空格
- 使用 UTF-8 编码保存文件

### 3. 检查号重复
**错误信息**: "检查号已存在"

**解决方案**:
- 检查 CSV 文件中是否有重复的检查号
- 检查数据库中是否已存在该检查号
- 系统会跳过重复的记录，继续导入其他数据

### 4. 必填字段缺失
**错误信息**: "必填字段缺失：XXX"

**解决方案**:
- 确保检查号、患者匿名ID、检查时间、检查部位字段不为空
- 检查 CSV 文件中是否有空单元格

### 5. 日期格式错误
**错误信息**: "检查时间格式错误，期望格式：yyyy-MM-dd HH:mm:ss"

**解决方案**:
- 使用正确的日期时间格式：`2024-03-01 10:30:00`
- 不要使用其他分隔符（如 `/` 或 `.`）

### 6. 年龄格式错误
**错误信息**: "年龄格式错误"

**解决方案**:
- 确保年龄字段为整数
- 如果年龄未知，可以留空

## 最佳实践

### 1. 数据准备
- 在导入前先验证数据的完整性和准确性
- 使用模板文件（`case_import_template.csv`）作为参考
- 建议先用小批量数据测试导入功能

### 2. 批量导入
- 单次导入建议不超过 1000 条记录
- 大量数据可分批导入，避免超时
- 导入前建议备份现有数据

### 3. 错误处理
- 仔细查看返回的错误列表
- 修正错误后可重新导入失败的记录
- 系统会自动跳过已存在的检查号

### 4. 性能优化
- 导入过程中避免并发操作
- 选择系统负载较低的时段进行大批量导入
- 导入完成后检查数据完整性

## 操作日志

所有批量导入操作都会记录在系统操作日志中，包括：
- 导入用户
- 导入时间
- 导入数量
- 操作结果

管理员可以在操作日志页面查询导入历史记录。

## 权限要求

- 需要系统管理员权限才能执行批量导入操作
- 普通医生用户无法访问此功能

## 技术支持

如遇到问题，请联系系统管理员或查看系统日志获取详细错误信息。
