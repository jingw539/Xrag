> 生成日期：2026-03-06

---

## 一、题目要求与系统定位

### 题目核心要求
"医学影像智能辅助诊断"要求系统能够：
- 对医学影像数据进行智能分析、快速读片、**报告生成**
- 利用 **计算机视觉、神经网络** 等技术进行影像诊断
- 实现 **病灶分析、参数量化** 功能
- 从海量影像中**快速发现微小病灶**、**直观定位病灶**
- 提升影像诊断质量，降低随访工作量

### 本系统定位
本系统聚焦 **胸部X光（CXR）智能辅助诊断**，通过 RAG（检索增强生成）+ 多模态大模型实现：
1. **AI报告自动生成**（Qwen-VL-Plus视觉理解 + DeepSeek文本处理）
2. **CheXpert 14类病理标签评测**（质量量化）
3. **相似病例检索**（RAG知识库辅助）
4. **术语规范化**（NLP标准化）
5. **全流程质控统计**（科研数据支撑）

### 2.3 AI集成（外部API）
- **报告生成**：调用 `dashscope.aliyuncs.com` 的 Qwen-VL-Plus 接口
  - 输入：MinIO图像URL + RAG参考病例上下文
  - 输出：JSON格式 findings/impression/confidence
- **CheXbert评测**：调用 DeepSeek Chat 接口
  - 输入：报告文本
  - 输出：14类标签 + F1/Precision/Recall/BLEU-4/ROUGE-L
- **术语分析**：调用 DeepSeek Chat 接口
  - 输入：报告文本
  - 输出：不规范术语列表 + 标准化建议

**当前模型进度：仅调用外部API接口，未本地部署模型。**

## 五、AI模型对接说明（给评审看）

### 当前实现（API调用模式）
```
前端 → Spring Boot → Qwen-VL-Plus API → 报告草稿
              ↘ DeepSeek API → CheXbert标签 + F1评分
              ↘ DeepSeek API → 术语规范化建议
```

### 预留的本地模型扩展路径
当具备GPU资源后，只需修改 `AiServiceClient` 中的调用地址，无需改动上层业务代码：
```
当前: https://dashscope.aliyuncs.com/... (Qwen-VL API)
替换: http://localhost:8000/report/generate (本地LLaVA-Med-LoRA)

当前: https://api.deepseek.com/... (DeepSeek API)  
替换: http://localhost:8001/eval/chexbert (本地CheXbert模型)
```

架构设计上已满足"统一接口+可替换实现"原则。
